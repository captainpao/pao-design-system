# Codebase Structure

## Directory Purposes

**src/components/:**
- Purpose: Reusable Web Component implementations
- Contains: One directory per component with implementation, styles, stories, and tests
- Key files: `[component-name].ts` (main), `[component-name].styles.ts`, `[component-name].stories.tsx`

**src/styles/:**
- Purpose: Design system tokens and foundational styling
- Contains: SCSS files for colors, typography, fonts, and base styles
- Key files:
  - `_colors.scss` - Color palette (primary, secondary, success, warning, danger, light, dark)
  - `_typography.scss` - Font sizes, weights, line heights
  - `_fonts.scss` - Custom font imports (VerlagSSm, FontAwesome)
  - `_base.scss` - Root-level defaults
  - `index.scss` - Central import point

**src/foundation/:**
- Purpose: Design documentation files for Storybook
- Contains: MDX files for Colors and Typography documentation
- Key files:
  - `Colors.mdx` - Color system documentation
  - `Typography.mdx` - Typography system documentation

**src/assets/:**
- Purpose: Static resources bundled with the library
- Contains:
  - `fonts/` - VerlagSSm custom font files (.woff2)
  - `images/` - Component documentation images

**.storybook/:**
- Purpose: Storybook configuration and customization
- Key files:
  - `main.ts` - Storybook configuration (stories location, addons, vite integration)
  - `manager.ts` - Manager UI customization
  - `preview.ts` - Preview settings
  - `preview-head.html` - Custom HTML in preview
  - `public/` - Static assets served by Storybook

**test/:**
- Purpose: Jest test files
- Contains: Component tests with Jest and @open-wc/testing
- Pattern: `[component].test.ts` for each component
- Key files:
  - `pao-button.jest.test.ts` - Jest tests with mocks
  - Other test files for different testing approaches

**dist/:**
- Purpose: Compiled output (generated by `npm run build`)
- Contains:
  - `pao-design-system.js` - ES module build
  - `pao-design-system.cjs` - CommonJS build
  - `pao-design-system.umd.js` - UMD build
  - `index.d.ts` - TypeScript declarations
  - `types/` - Generated .d.ts files for all components
  - `assets/` - Bundled fonts and images

## Key File Locations

**Entry Points:**
- `src/index.ts` - Library main entry point, imports styles and exports all components
- `.storybook/main.ts` - Storybook development server entry

**Configuration:**
- `tsconfig.json` - TypeScript compiler settings (ES2020 target, strict mode)
- `vite.config.ts` - Vite build configuration (dual build: ES + CJS, Lit excluded)
- `jest.config.js` - Jest test runner configuration (jsdom environment, ts-jest)
- `.eslintrc.json` - ESLint rules (TypeScript, Prettier, Storybook plugins)
- `.prettierrc` - Code formatter settings (single quotes, 100 char line width)

**Core Logic:**
- `src/components/pao-button/pao-button.ts` - Primary button component
- `src/components/pao-button-group/pao-button-group.ts` - Button group with selection
- `src/components/pao-icon/pao-icon.ts` - Icon component with FontAwesome integration

**Styling:**
- `src/styles/index.scss` - Central style import (ensures fonts load)
- `src/components/[component]/[component].styles.ts` - Per-component styles (Lit CSS)

**Testing:**
- `test/pao-button.jest.test.ts` - Jest tests with mocks
- Component stories contain interactive examples but are not unit tests

## Naming Conventions

**Files:**
- Component implementations: kebab-case.ts (e.g., `pao-button.ts`)
- Style files: [name].styles.ts (e.g., `pao-button.styles.ts`)
- Story files: [name].stories.tsx (e.g., `pao-button.stories.tsx`)
- Test files: [name].test.ts or [name].jest.test.ts
- Utils: [name].utils.ts (e.g., `pao-icon.utils.ts`)
- SCSS: Underscore-prefixed for partials (e.g., `_colors.scss`)

**Directories:**
- Component directories: kebab-case matching custom element name (e.g., `pao-button/`)
- Feature directories: lowercase (e.g., `components/`, `styles/`, `assets/`)

**Components (Custom Elements):**
- Custom element tags: `pao-[component-name]` (e.g., `pao-button`, `pao-icon`)
- Class names: PascalCase (e.g., `PaoButton`, `PaoIcon`)
- CSS classes in templates: kebab-case (e.g., `.pao-button`, `.group-button`)

**Properties & Events:**
- Component properties: camelCase (e.g., `variant`, `disabled`, `pillShape`)
- Custom events: camelCase with "pao" prefix (e.g., `paoClick`, `pao-selection-change`)
- CSS custom properties: kebab-case with `--pao-` prefix (e.g., `--pao-color-primary`)

## Where to Add New Code

**New Component:**
1. Create directory: `src/components/pao-[name]/`
2. Create files:
   - `pao-[name].ts` - Component implementation (extends LitElement, use @customElement)
   - `pao-[name].styles.ts` - Styles using Lit css template literal
   - `pao-[name].stories.tsx` - Storybook stories with controls
   - `pao-[name].ts` documentation in component or separate .mdx
3. Export from `src/index.ts`: `export * from './components/pao-[name]/pao-[name]'`
4. Add tests in `test/pao-[name].test.ts`

**New Style Variable:**
- Colors: Add to `src/styles/_colors.scss` under appropriate section
- Typography: Add to `src/styles/_typography.scss`
- Fonts: Import in `src/styles/_fonts.scss`
- All: Reference via CSS custom properties in components

**New Utility:**
- Location: `src/components/[component]/[component].utils.ts`
- Export functions or data maps
- Import in component implementation file

**Shared Helpers:**
- Currently no shared utilities directory
- Suggested: Create `src/utils/` for cross-component helpers if needed

## Special Directories

**dist/:**
- Purpose: Build output
- Generated: Yes (via `npm run build`)
- Committed: No (.gitignored)
- Contents: Bundled JS, type declarations, assets

**storybook-static/:**
- Purpose: Built Storybook site
- Generated: Yes (via `npm run build-storybook`)
- Committed: No (.gitignored)
- Contents: Static HTML, CSS, JS for deployed Storybook

**node_modules/:**
- Purpose: Installed dependencies
- Generated: Yes (via `npm install`)
- Committed: No (.gitignored)

**coverage/:**
- Purpose: Jest coverage reports
- Generated: Yes (via `npm run test:coverage`)
- Committed: No (.gitignored)

**test/:**
- Purpose: Test files (different from component-colocated tests)
- Generated: No
- Committed: Yes
- Usage: Jest runs tests from here matching `**/*.test.ts` pattern
